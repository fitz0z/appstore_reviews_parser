# Contributing to App Store Reviews Parser

This document provides guidelines for contributors.

## How to Contribute

### Reporting Bugs

Before opening a bug report, please check if the issue has already been reported.

When filing a bug report, please include:

1. A clear and descriptive title
2. Steps to reproduce the issue
3. Expected behavior
4. Actual behavior
5. Environment details (PHP version, OS, etc.)
6. Any relevant error messages or logs

### Suggesting Enhancements

Enhancement suggestions are welcome! Please provide:

1. A clear description of the enhancement
2. Use cases for the enhancement
3. Any implementation ideas (optional)

### Pull Requests

1. Fork the repository
2. Create a feature branch from the `main` branch
3. Make your changes
4. Add tests if applicable
5. Ensure all tests pass
6. Commit your changes with a clear message
7. Push to your fork and submit a pull request

## Coding Standards

- Follow PSR-12 coding standards
- Add documentation for new methods
- Keep code simple and readable
- Add appropriate type hints
